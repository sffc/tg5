## Abstracts

### Safe Until It Isn’t: Reclaiming Isolation in JavaScript Systems (J. M. Gómez Moreno)

Modern JavaScript runtimes provide strong isolation guarantees
at the language level, leveraging memory and type safety to
protect applications from a wide class of vulnerabilities.
However, this security model breaks down when native
components—such as binary add-ons or subprocesses—are
introduced. These components operate outside the managed
runtime, often with unrestricted access to system resources,
effectively bypassing the protections that make JavaScript
environments attractive in the first place.
This talk argues that robust sandboxing for JavaScript
applications must extend beyond the language boundary
and incorporate mechanisms rooted in both hardware and
operating system abstractions. We explore how hardware
primitives, such as protection keys, can be combined with
OS-level enforcement techniques to confine untrusted native
code with fine granularity. Building on this foundation,
we present a systematic approach based on system call
interposition, enabling comprehensive mediation of resource
accesses without requiring invasive changes to application code.
We then discuss our recent work on hybrid interposition
mechanisms, which combine kernel support with user-space
techniques to achieve both strong security guarantees and
practical performance. Through this lens, we highlight the
challenges of building deployable sandboxes—such as completeness,
efficiency, and robustness—and outline how modern designs can
overcome long-standing limitations. Together, these ideas point
toward a principled path for securing JavaScript ecosystems
in the presence of native code, bridging the gap between
high-level safety guarantees and low-level system realities.

### Verified Interpreters for Dynamic Languages with Applications to the Nix Expression Language (R. Krebbers)

To study the semantics of a programming language, it is useful
to consider different specification forms—e.g., a substitution-based
small-step operational semantics and an environment-based
interpreter—because they have mutually exclusive benefits. Developing
these specifications and proving correspondences is challenging for
"dynamic"/"scripting" languages such as JavaScript, PHP and Bash. We
study this challenge in the context of the Nix expression language, a
dynamic language used in the eponymous package manager and operating
system. Nix is a Turing-complete, untyped functional language designed
for the manipulation of JSON-style attribute sets, with tricky features
such as overloaded use of variables for lambda bindings and attribute
members, subtle shadowing rules, a mixture of evaluation strategies, and
tricky mechanisms for recursion.

We show that our techniques are applicable beyond Nix by starting from
the call-by-name lambda calculus, which we extend to a core lambda
calculus with dynamically computed variable names and dynamic binder
names, and finally to Nix. Our key novelty is the use of a form of
deferred substitutions, which enables us to give a concise
substitution-based semantics for dynamic variable binding. We develop
corresponding environment-based interpreters, which we prove to be sound
and complete (for terminating, faulty and diverging programs) w.r.t. our
operational semantics based on deferred substitutions.

We mechanize all our results in the Rocq prover and showcase a new
feature of the Rocq-std++ library for representing syntax with maps in
recursive positions. We use Rocq's extraction mechanism to turn our Nix
interpreter into executable OCaml code, which we apply to the official
Nix language tests. Altogether this gives rise to the most comprehensive
formal semantics for the Nix expression language to date.

This is joint work with [Rutger Broefhoff](https://fautchen.eu/), and has been [published at ICFP
2025](https://dl.acm.org/doi/10.1145/3747537).
